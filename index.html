skills and competencies

HTML5

Like many others, HTML is where it all started for me.

I spend my working days between generating the modern, semantic markup for the websites we create in-house. And then a few times a week, I have to throw the modern approach out of the window and put on a different hat for creating html e-mail templates from scratch, ensuring they look clean for Microsoft Outlook email clients, as well as responsive on modern mobile devices.

CSS3 & Sass

Sass is a great way to make CSS easier to maintain, and was a good progression point once I felt comfortable working with CSS3.

I strive for a modular, clean and re-usable Sass architecture and keep things clean and team-mate friendly by insisting upon a commonly agreed upon naming convention.

Of all the joy that Sass brings with it, I'd say the pre-processor features I have the most experience with are; variables, mixins and nesting.

ECMAScript

Javascript is another one of my strongest skills as a Developer. I'm keen to produce client-side code that not only works, but is maintainable. Exploring modern JavaScript was the hook that really encouraged me want to pursure web development as a career path as opposed to just a hobby.

In my current role as a web developer, I've updated our development team's JavaScript guidelines to move away from jQuery, and jQuery plug-ins, and to modern, es6 syntax. I've introduced the team to lightweight, modern javascript libraries such as GSAP, scroll magic and Rellax for brochure website projects.

I've got professional work experience supporting an Angular codebase, decoupled from an Laravel API.

PHP: Hypertext Processor

The server-side language of choice at most studios I have spent time working for is PHP. Some love it and some hate it. Persoanlly, I think it's a good tool.

A misunderstood C API for the web, Modern PHP is fine to work with on small to medium sized projects. With modern frameworks like laravel taking the industry by storm, it's also now used more and more on large scale projects.

I've got plenty of experience working in WordPress and Laravel, building WordPress sites from scratch, as well as longer term maintenance work such as small modifications, framework or theme/plugin upgrades in both `ecosystems`.

WordPress

WordPress powers around 1/3 of the web now, and all agencies where I have spent some time have used it as their main platform for new brochure websites.

In my curent role, I take on a majority of the WordPress development and maintenance projects. I create custom themes from scratch, as well as small plugins.

WordPress may not be the solution to every brief, but its flexible, secure and boasts a large and dedicated open-source community, whilst offering great value to it's many users.

Gulp

WordPress theme develpoment needs to be done right. Its vital for reaching a balance between a good page load time and a maintainable client-side codebase

Every team should consider a build-tool like gulp to help automate tasks such as Sass and ES6+ Javascript compilation, and minification, image optimising, and the many other useful funcitons you can levearge a build-tool like gulp for.

We're currently working with Gulp 4 on new projects, and most often Gulp 3 on legacy code bases.

Responsive process

Today, making websites responsive is a standard expectation not only just by clients, but also by search engines if they want their site to do well.

I've been taught to take a thourough approach to responsive testing, ensuring that projects are tested on various mobile devices, as opposed to just the broswer's developer tools.

Cross-Browser testing

Like everyone who has ever worked as a Developer, I'm glad we're not having to maintain sites for IE anymore, but this doesn't mean that cross-browser testing isn't still an essential aspects of building a web-based product.

I'm well aware of the importance of cross-broswer testing, and can sometimes anticipate the pitfalls associated with using modern CSS, or Javascript techniques in older broswers.

Photoshop

Images can make or break a website, and so I decided early on that working with image editors like Photoshop or GNU was a worthwhile tool to have in the box.

In my curent role, I use photoshop for both web and email development tasks, from precsion cut-outs, to template mock-ups.

Whilst photoshop is a skill-area in and of itself, I appreciate the transferable principles such as working with layers, and masks that means we can happily jump into another editing suite like GNU-IMP, and still have a fair command of the interface.

Maintainance

Initially I spent lots of time teaching myself about creating websites, but when starting to gain active work experience I began to gain an understanding of how a website is maintained.
